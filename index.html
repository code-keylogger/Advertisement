CodeLogger

           

            How do people come up with solutions to programming questions and math proofs? It may seem like a very individual process, but is it possible to replicate this or find patterns in how different people code? Currently, there is very little research into this. CodeLogger is the solution to fixing this lack of information.

VSCode (Visual Studio Code) is one of the most popular integrated development environments (IDE) in the world and with CodeLogger, it is possible to track what people type while creating solutions and writing programs using it. This extension for the IDE keeps track of the programmers’ movements, collecting their every movement and storing it. This data can be analyzed and will be used to study how exactly people think when they create solutions. Using CodeLogger, it is also possible to play back precisely what each programmer was typing.

CodeLogger is a straightforward extension that can be installed onto the VSCode platform. Once installed, this extension simply keeps track of all edits you make to whichever file is currently in use. All these edits are stored in a MongoDB database, which can then be fetched via an API and converted into a Panda’s Data frame (python object that can be used for analysis). The extension will also provide a playback feature for researchers as well as all users to see what exactly was typed and provide a visualization of each programmers thought process and output using various python scripts. The visualizer will take existing data in the database and replay the edits stored at normal speed, with the ability to pause, play, and scrub playback.

If you’re interested in researching this topic CodeLogger is perfect for you. Our product has unlimited potential for training models and artificial intelligence.

CodeLogger will be released for use on December 8th, 2022. It was created in partnership with Professor Katz to assist him with his research papers. Therefore it will be owned by Professor Garrett Katz for use in his research. Professor Katz will be using the data collected in his research on NLP (Natural Language Processing). Natural Language Processing is the field of Artificial Intelligence that deals with human language. This data could potentially be used to train models, AI algorithms trained on a set of data, and create an LLM (Large Language Model) that could solve problems fed into it, by “mimicking” the tested correct inputs of the users that trained the model.

Data collection will be done using volunteers selected by Professor Katz, who will be asked to solve a coding problem or a proof. These volunteers will be given a problem and asked to write their code in a specified area. While solving these problems, the volunteers will be given automatic test results to help guide them. All edits they make while solving these problems will be captured by CodeLogger and Professor Katz will have access to all of it using a secure REST (Representational state transfer) API.
